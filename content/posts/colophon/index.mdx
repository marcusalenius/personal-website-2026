## Design

The design of this website is inspired by [Gabriel Sha's](https://gsha22.github.io/), [Ben Cuan's](https://bencuan.me/), [Nikhil Jha's](https://nikhiljha.com/), and [Oliver Ni's](https://oliver.ni/) websites. It is typeset in [Nunito Sans](https://fonts.google.com/specimen/Nunito+Sans) and `code` is in [Inconsolata](https://fonts.google.com/specimen/Inconsolata?preview.script=Latn). The icons are [Phosphor Icons](https://phosphoricons.com/).

## Tech Stack

It's a Next.js and React site written in TypeScript that is statically exported and served via GitHub Pages. It auto-deploys on every push to `main`. The source code can be found [here](https://github.com/marcusalenius/personal-website-2026).

The content lives in a `content/` directory and is processed by [Velite](https://velite.js.org/), a tool that turns content files into typed, validated data collections. The entries on the `/work` page live in `content/work/*.yml` which get compiled into a JSON dataset by Velite at build time, and the page component imports that data, sorts it, and renders it.

Posts are written in MDX (an extension of Markdown that allows me to add custom react components into the markup). To create a new post, I add the directory `content/posts/[slug]` with an `index.mdx` file, and optionally a `media/` subdirectory with any media for the post. The MDX file has frontmatter and Markdown body:

```md
---
```
